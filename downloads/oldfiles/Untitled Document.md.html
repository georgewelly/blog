“Problem Solving and Being Lazy”

I’ve been sorted into group two, first presentation is “Problem Solving and Being Lazy”, led by Billy Mayor.This presentation was all about software engineering, how to manage problems and making them efficient (“being lazy”). Why write code to do a job multiple times when one could write a function and save 20 lines of code and time as well? When problem solving, we should treat all our problems with a to-do list like RAM, when we stop doing work, we should put our remaining problems into swapspace, or on a piece of paper. This makes for very efficient problem solving.
